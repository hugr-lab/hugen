# Queries

> OPTIONAL. Filled in by the researcher when it can offer a verified
> starter query or a check the checker can recompute. Leave the
> sections empty (or write "none") when there is nothing useful to
> hand down. Record query SHAPES only — never live result values.
>
> **VALIDATE every query with `data-validate_graphql_query` BEFORE
> you record it** — fix until it passes; if you can't make it pass,
> write "none". A guessed, wrong-syntax shape misleads the worker —
> worse than nothing. Do NOT trust your prior for hugr grammar: it is
> non-standard — aggregations group via `key { <field> }` and nest
> the aggregates under `aggregations { _rows_count … }` (there is NO
> `group_by:` argument). Validation is the only proof.
