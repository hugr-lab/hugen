## Provider matrix

| Provider          | URL form                              | Secret setup (one statement)                                                            |
|-------------------|---------------------------------------|------------------------------------------------------------------------------------------|
| AWS S3            | `s3://bucket/path`                    | `CREATE SECRET (TYPE S3, PROVIDER credential_chain);`                                    |
| Cloudflare R2     | `r2://bucket/path` or `s3://...`      | `CREATE SECRET (TYPE R2, PROVIDER credential_chain);`                                    |
| GCS               | `gs://bucket/path` or `gcs://...`     | `CREATE SECRET (TYPE GCS, PROVIDER credential_chain);`                                   |
| MinIO / custom    | `s3://...` against a custom endpoint  | `CREATE SECRET (TYPE S3, KEY_ID '...', SECRET '...', ENDPOINT '...', USE_SSL true);`     |
| Azure             | `az://...`, `azure://...`, `abfss://` | `LOAD azure; CREATE SECRET (TYPE AZURE, PROVIDER credential_chain);`                     |
| Public bucket     | any                                   | none — DuckDB anonymous-reads when no secret is registered                               |
| HTTPS only        | `https://...` / `http://...`          | none — `httpfs` handles plain HTTP(S) GETs                                               |

The credential-chain providers (`credential_chain`, `config`) read from
the same env / config file the upstream provider's tooling uses
(`AWS_ACCESS_KEY_ID`, `~/.aws/credentials`, GCS service-account JSON, …).
Pre-set those in the operator's environment if the LLM should not see
explicit keys.

Secrets persist for the session under
`${SESSION_DIR}/.duckdb/secrets/` and die when the session closes.

## Listing a bucket / prefix

```sql
SELECT filename,
       (size / 1024 / 1024)::DECIMAL(10,1) AS size_mb,
       last_modified
FROM read_blob('s3://bucket/data/*')
ORDER BY filename
LIMIT 50;
```

Project only `filename`, `size`, `last_modified` — selecting `content`
would download every object. For large prefixes, narrow the glob first.

## Previewing a file

```sql
DESCRIBE FROM 's3://bucket/data/file.parquet';
SELECT count(*) AS row_count FROM 's3://bucket/data/file.parquet';
FROM 's3://bucket/data/file.parquet' LIMIT 20;
```

Predicate pushdown applies to remote Parquet — `WHERE` clauses against
indexed columns avoid downloading the rest.

## Parquet metadata without payload

```sql
SELECT file_name,
       sum(row_group_num_rows) AS total_rows,
       (sum(row_group_compressed_bytes) / 1024 / 1024)::DECIMAL(10,1) AS compressed_mb
FROM parquet_metadata('s3://bucket/data/*.parquet')
GROUP BY file_name;
```

This reads only the footer of each Parquet file — orders of magnitude
cheaper than scanning rows for a count.

## Public datasets

Some buckets serve anonymous reads (Overture Maps, AWS open-data,
NOAA). For those, omit the secret entirely:

```sql
FROM 's3://overturemaps-us-west-2/release/.../poi/*.parquet' LIMIT 5;
```

If a public read fails with `403`, the bucket may require a region hint
in the secret:

```sql
CREATE SECRET (TYPE S3, PROVIDER config, REGION 'us-west-2');
```

## Common errors

| Symptom                              | Fix                                                                                  |
|--------------------------------------|--------------------------------------------------------------------------------------|
| `HTTP Error: 403`                    | Credentials missing / wrong; for public buckets, set the region in the secret       |
| `HTTP Error: 404`                    | URL or bucket name typo; check region / endpoint                                     |
| `IO Error: Connection ...`           | Endpoint unreachable from this host; check operator firewall                         |
| `Failed to glob ...`                 | Glob refers to a prefix DuckDB cannot list — narrow the wildcard                     |

## Saving a remote query locally

`COPY` writes are local by default; combining a remote `read_parquet`
with a local `COPY ... TO 'data/...'` is the canonical "ingest" idiom:

```sql
COPY (
  SELECT * FROM read_parquet('s3://bucket/data/2026/*.parquet')
  WHERE event_date >= DATE '2026-04-01'
) TO 'data/april.parquet';
```
